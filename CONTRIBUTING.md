# Contributing to B.Y.T.E.

B.Y.T.E. is in the design phase. At this stage, **conceptual contributions only** — no code will be accepted until the core thesis is validated.

## How to Contribute

### Open Issues

- Discuss architecture decisions
- Challenge assumptions in the core thesis
- Suggest improvements to subsystem designs
- Propose evaluation methodology refinements

### Pull Requests

Documentation changes only:

- Fix errors or inconsistencies in existing docs
- Add clarity to underspecified sections
- Expand on known gaps with proposed solutions
- Improve diagrams or cross-references

## Where to Start

- [KNOWN_GAPS.md](docs/core/KNOWN_GAPS.md) — the most open-ended areas
- [DESIGN_DECISIONS.md](docs/core/DESIGN_DECISIONS.md) — rationale you can challenge
- [EVALUATION.md](docs/core/EVALUATION.md) — methodology you can refine
- [DOCUMENT_MAP.md](docs/DOCUMENT_MAP.md) — understand the full structure

## What's Not Accepted (Yet)

- Code implementations
- New subsystems not aligned with the existing architecture
- Changes to the core thesis without engagement with the evaluation methodology

## The Core Thesis

This project tests a single hypothesis: **can accumulated structure substitute for repeated inference?** Contributions should engage with this question directly — either by strengthening the mechanism, improving the evaluation, or identifying why it might fail.
